# 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명:** 로또 번호 분석 및 추천 시스템
- **목표:** 역대 로또 당첨 데이터를 기반으로 통계적 지표를 분석하고, 사용자에게 과학적인 번호 조합을 추천하는 웹 서비스 제공.
- **핵심 가치:** 단순 무작위 생성이 아닌, 통계적 확률이 높은 구간 내의 번호를 사용자에게 제안하여 흥미와 분석적 재미를 제공함.

## 2. 타겟 사용자
- 로또를 주기적으로 구매하며 통계적 접근에 관심이 있는 사용자.
- 무작위 번호 자동 선택보다 특정 패턴이나 필터링이 적용된 번호를 선호하는 사용자.
- 역대 당첨 번호의 빈도 및 통계를 시각적으로 확인하고 싶은 사용자.

## 3. 핵심 기능 (Features)
### 3.1. 번호 생성 및 실시간 분석
- **번호 생성:** 버튼 클릭 시 1~45 사이의 숫자 6개를 중복 없이 생성.
- **실시간 통계 분석:** 생성된 번호에 대해 즉시 다음과 같은 지표 분석.
    - 홀짝 비율, 고저 비율, 총합(Sum).
    - 직전 회차 이월수 및 주변수 포함 여부.
    - AC값(산술적 복잡도), Span(최대-최소 차이).
    - 소수, 합성수, 3/5배수, 쌍수, 완전제곱수 포함 개수.
    - 특수 패턴(모서리 패턴, 삼각형 패턴) 분석.
- **상태 표시:** 각 지표가 통계적으로 '최적(Optimal)', '보통(Normal)', '주의(Warning)' 범위에 있는지 시각적으로 표시.

### 3.2. 통계 시각화 (Statistics)
- 역대 당첨 번호의 출현 빈도 확인.
- 구간별 출현 횟수 및 미출현 번호 데이터 제공.
- 홀짝, 고저, 총합 등의 역대 분포 그래프/표 제공.

### 3.3. 상세 분석 (Combination/Advanced Analysis)
- 특정 조건을 설정하여 번호를 필터링하거나 정밀 분석하는 기능.
- 회차별 상세 당첨 데이터 조회.

### 3.4. 사용자 편의 기능
- **이용 가이드:** 서비스 사용 방법 및 각 통계 지표에 대한 설명 제공.
- **히스토리:** 최근 생성한 번호 내역 저장 및 확인 (LocalStorage 활용).
- **반응형 디자인:** 모바일 및 데스크탑 환경 최적화.

## 4. 비기능 요구사항
- **성능:** 데이터 로드 및 번호 생성/분석이 1초 이내에 완료되어야 함.
- **보안:** 개인정보를 수집하지 않으며, 로컬 스토리지를 활용한 데이터 저장.
- **SEO:** 검색 엔진 최적화를 위한 메타 태그 및 구조화된 콘텐츠 제공.

## 5. 성공 지표
- 페이지 뷰(PV) 및 번호 생성 버튼 클릭 수.
- 재방문율 (사용자가 통계 업데이트를 확인하기 위해 다시 방문하는지 여부).
