# PRD (Product Requirement Document) - v4.0

## 1. 프로젝트 비전
역대 당첨 데이터를 기반으로 과학적이고 정밀한 통계 지표를 제공하며, 전문가급 데이터 가시화와 AI 예측 모델을 통해 사용자의 로또 조합 결정을 돕는 프리미엄 분석 플랫폼.

## 2. 핵심 타겟 유저
- 데이터 기반의 논리적 조합을 선호하는 로또 분석가.
- 복잡한 통계를 한눈에 파악하고 싶은 파워 유저.
- 실제 로또 용지의 감성을 디지털에서 느끼고 싶은 사용자.

## 3. 주요 기능 요구사항
### 3.1. 지능형 데이터 분석 (G0 ~ G6)
- **G0 시너지**: 지표 간 상관관계를 분석하여 모순된 조합을 필터링.
- **G1-G5 표준 지표**: 총합, 홀짝, 분포 등 22개 기본 통계 지표 제공.
- **G6 고급 지표**: 첫 수/끝 수 범위, 평균 간격 등 심층 분석 지표 추가.

### 3.2. 전문가급 UI/UX (v4.0 고도화)
- **45도 대각선 헤더**: 좁은 공간에 다수의 지표를 배치하여 정보 밀도 극대화.
- **엑셀형 틀 고정 (Sticky)**: 가로 스크롤 시에도 회차와 당첨번호를 고정하여 대조 편의성 제공.
- **리얼 페이퍼 마킹**: 실제 용지 질감 구현 및 직관적인 컬러 강조 효과로 아날로그 감성 이식.
- **10열 정밀 그리드**: 예측 번호 풀을 10열 단위로 정렬하여 시각적 인지 속도 향상.

### 3.3. AI 예측 및 백테스트
- **다차원 스코어링**: 빈도, 미출현 기간(Gap), 이웃수, 이월수 가중치 모델.
- **블라인드 백테스트**: 분석 시점 기준 미래 데이터를 차단한 정밀한 성과 검증 엔진.
- **등급제 성과 지표**: 적중 실적을 S/A/B 등급으로 시각화.

## 4. 기술 스택 및 정책
- **Frontend**: Vanilla JS (ES6+), CSS3 (Flex/Grid/Sticky), SVG (Charts).
- **Backend (Pipeline)**: Python (Data Scraping & Analysis).
- **Architecture**: `indicators.js` 중심의 설정 기반 동적 렌더링 (SSOT).
- **Standard**: Toss/KakaoBank 스타일의 미니멀 핀테크 디자인 시스템 준수.
